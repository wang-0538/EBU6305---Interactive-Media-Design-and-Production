# 导航与页脚：两种打开方式的同步指南

本站导航栏和页脚有两种加载方式。修改 **`components/navbar.html`** 或 **`components/footer.html`** 之后，若不同步，两种打开方式看到的内容可能不一致。本说明告诉你原因、以谁为准，以及如何保持同步。

---

## 一、两种打开方式分别是什么？

| 打开方式 | 典型操作 | 浏览器地址 |
|----------|----------|------------|
| **方式 A：本地 HTTP 服务** | VS Code Live Server、`npx serve` 等 | `http://127.0.0.1:…` 或 `http://localhost:…` |
| **方式 B：直接打开文件** | 双击 `index.html` | `file:///…/index.html` |

---

## 二、为什么需要「同步」？

项目里的 **`js/main.js`** 会：

1. **先尝试** 用 `fetch()` 请求 **`components/navbar.html`** 和 **`components/footer.html`**（相对路径）。
2. 若请求失败（**方式 B** 下浏览器通常会拦截本地文件的 `fetch`），则改用 **`main.js` 里内嵌的 HTML 字符串**（`EMBEDDED_NAVBAR`、`EMBEDDED_FOOTER`）作为备用。

因此：

- **方式 A**：你看到的是 **磁盘上的 `components/*.html`**（改文件即生效）。
- **方式 B**：你看到的是 **`main.js` 里写死的那份备份**（除非已与 `components` 对齐）。

**只改 `components`、不更新 `main.js` 内嵌** → 方式 A 新、方式 B 旧。  
**要保证两种打开方式一致** → 改完 `components` 后，必须更新 `main.js` 中的内嵌片段（见下文脚本）。

---

## 三、以谁为准？（建议）

- **唯一维护源（真源）**：**`components/navbar.html`**、**`components/footer.html`**。  
- **`main.js` 里的 `EMBEDDED_*`**：视为 **`components` 的镜像**，专门给 **方式 B** 使用；应用脚本从 `components` 自动生成，避免手抄出错。

---

## 四、如何同步？（推荐：自动脚本）

已提供脚本：**`scripts/sync-fragments.mjs`**。它会读取上述两个 HTML 文件，并写回 **`js/main.js`** 中 `// --- BEGIN AUTO-GENERATED ---` 与 `// --- END AUTO-GENERATED ---` 之间的内容。

### 前置条件

- 本机已安装 [Node.js](https://nodejs.org/)（安装后在终端执行 `node -v` 有版本号即可）。

### 操作步骤

1. 用编辑器修改 **`components/navbar.html`** 或 **`components/footer.html`**，保存。
2. 打开终端，**进入项目中的 `color-learning-website` 目录**（与 `README.md` 同级、内含 `js`、`components` 的那一层）。
3. 执行：

   ```bash
   node scripts/sync-fragments.mjs
   ```

4. 终端若提示成功，说明 **`js/main.js`** 已更新。请将 **`components` 的修改** 与 **`js/main.js` 的修改** 一并保存、提交或打包发给他人。

### 提交 / 发给别人前检查清单

- [ ] 已修改 `components`（若本次有改导航或页脚）。
- [ ] 已运行 `node scripts/sync-fragments.mjs`（若本次有改导航或页脚）。
- [ ] `git` 提交或压缩包中 **同时包含** 更新后的 `components` 与 `js/main.js`。

---

## 五、没有 Node.js 时怎么办？

若暂时无法运行脚本，只能 **手动** 把 `components/navbar.html`、`components/footer.html` 的 HTML **等价复制** 到 `main.js` 里 `EMBEDDED_NAVBAR`、`EMBEDDED_FOOTER` 对应位置（注意引号与换行，易出错）。**仍建议安装 Node 后改用脚本**，省时且不易漏改。

---

## 六、协作与验收建议

| 场景 | 建议 |
|------|------|
| **日常开发** | 优先用 **方式 A（Live Server）**，改 `components` 即可立即预览。 |
| **要求对方「双击也能看」** | 交付前务必执行 **第四节同步脚本**，并让对方拿到 **已同步的 `main.js`**。 |
| **只要求课堂演示用 Live Server** | 仍建议在每次改导航/页脚后运行同步脚本，避免自己偶尔用方式 B 调试时看到旧版。 |

---

## 七、相关文件一览

| 文件 | 作用 |
|------|------|
| `components/navbar.html` | 导航 HTML（真源） |
| `components/footer.html` | 页脚 HTML（真源） |
| `js/main.js` | `fetch` 加载上述文件；失败时使用 `EMBEDDED_*` |
| `scripts/sync-fragments.mjs` | 从 `components` 生成 `main.js` 内嵌片段 |

如有疑问，可对照 **`js/main.js` 文件顶部的英文注释**（其中也说明了同步命令）。
